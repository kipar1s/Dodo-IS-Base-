## 📋 Описание проекта

Настольное приложение для управления заказами пиццерии с использованием Entity Framework Core и базы данных Microsoft Access.

Система позволяет:
- Создавать, редактировать и удалять заказы.
- Выбирать клиента из справочника.
- Добавлять в заказ несколько товаров с указанием количества.
- Отслеживать состав товаров из ингредиентов.
- Назначать ответственного сотрудника и способ оплаты.
- Управлять курьерами (номер ВУ, номер машины).
- Хранить данные о клиентах, товарах, ингредиентах, сотрудниках и квалификациях.

## ⚙️ Установка и запуск

### Требования

- Windows 10/11
- Microsoft .NET 10.0 SDK (или Visual Studio 2022)
- Microsoft Access Database Engine (для работы с .accdb)
- Visual Studio 2022 (рекомендуется)

### Сборка и запуск

1. **Клонируйте репозиторий:**
   ```bash
   git clone https://github.com/kipar1s/dodoEF.git
   ```

2. Откройте `dodoEF.sln` в Visual Studio.

3. Восстановите пакеты NuGet:
   ```bash
   dotnet restore
   ```

4. При необходимости отредактируйте строку подключения в `ApplicationDbContext.cs`:
   ```csharp
   optionsBuilder.UseJet(@"Provider=Microsoft.ACE.OLEDB.12.0;Data Source=путь_к_dodoDb2.accdb");
   ```

5. Примените миграции (создаст БД, если её нет):
   ```bash
   dotnet ef database update
   ```

6. Нажмите **F5** для запуска.

## 🗄 База данных

База данных `dodoDb2.accdb` (Microsoft Access) создаётся автоматически при первом запуске через `Database.EnsureCreated()`.

### Основные сущности и связи

| Сущность | Назначение | Ключевые связи |
|:---|:---|:---|
| **Client** | Клиенты | один ко многим с `Oder` |
| **Oder** | Заказы | один к одному с `Plateg`, многие ко многим с `Tovar` (через `OderTovar`) |
| **Tovar** | Товары | многие ко многим с `Ingr` (через `TovarIngr`) |
| **Ingr** | Ингредиенты | многие ко многим с `Tovar` |
| **Plateg** | Способы оплаты | один к одному с `Oder` |
| **Personal** | Сотрудники | один ко многим с `Oder`, многие ко многим с `Evelobilitile` |
| **Curier** | Курьеры | наследник `Personal`, добавляет `Num_VY` и `Num_Car` |
| **Categori** | Категории сотрудников | один ко многим с `Personal` |
| **Evelobilitile** | Уровни квалификации | многие ко многим с `Personal` |

## 🖥 Реализованные формы

### Главная форма (`DodoMainForm`)

- MDI-интерфейс с меню.
- Ролевая модель доступа (администратор, менеджер, кассир).
- Пункты меню: Справочники, Перечисления, Заказы.

### Список заказов (`OderListForm`)

- Таблица заказов с полями: ID, клиент, дата, сумма, статус, адрес.
- Кнопки: **Создать**, **Редактировать**, **Удалить**, **Обновить**.

### Форма заказа (`OderFormUnit`)

- Выбор клиента через диалог.
- Добавление/удаление товаров (два грида: товары в заказе + все товары).
- Выбор способа оплаты и статуса платежа (ListView с CheckBoxes).
- Выбор статуса заказа («Доставлено» / «Отменен»).
- Автоматический расчёт суммы.

### Список сотрудников (`PersonalListForm`)

- Таблица сотрудников с фильтрацией по категориям.
- Кнопки: **Создать**, **Редактировать**, **Удалить**.

### Список курьеров (`CurierListForm`)

- Таблица курьеров (фильтр по категории `categoriId == 21`).
- Поля: ФИО, стаж, номер ВУ, номер машины.
- Кнопки: **Создать**, **Редактировать**, **Удалить**.

### Справочники-перечисления (`EnumForm`)

- `CategoriEnumForm` — категории сотрудников.
- `EvelobEnumForm` — уровни квалификации.
- `IngrEnumForm` — ингредиенты.
- `PlategEnumForm` — способы оплаты.

## 📦 Инициализация тестовыми данными

Метод `SeedDatabase` в `DodoMainForm` заполняет таблицы демонстрационными данными:

- 5 клиентов
- 6 способов оплаты
- 4 категории сотрудников
- 6 уровней квалификации
- 6 товаров
- 6 ингредиентов
- 5 сотрудников + 3 курьера
- Связи товаров и ингредиентов (состав блюд)
- 5 заказов со связями к клиентам, платежам, сотрудникам и товарам

Для заполнения вызовите метод через меню **«Проверка БД»** в главной форме.

> ⚠️ Рекомендуется использовать в разработке, не в production.

## 🔄 Миграции Entity Framework Core

Проект использует миграции для синхронизации модели и базы данных.

### Создание новой миграции
```bash
dotnet ef migrations add NameOfMigration
```

### Применение миграций к БД
```bash
dotnet ef database update
```

### Генерация идемпотентного SQL-скрипта
```bash
dotnet ef migrations script --idempotent
```
